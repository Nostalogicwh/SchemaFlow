# v0.5.2
## 功能修复
1. 工作流执行，无法停止，两个按钮，需要强制取消
2. 进入页面会调用两次相同的workflow接口
3. 用户干预节点，前台模式是一个遮罩弹窗，影响操作，点击继续后，在最右侧监控栏出现了继续的提示；后台模式无提示；点击继续无反应
4. 拖拽功能异常，拖动日志栏，会影响实时截图的上部，期望效果是拖拽哪里，哪里移动，其他位置不相对移动
5. 登录信息留存，前一个已经登陆过的工作流，下一次运行还是需要登录
## 优化
1. AI定位，去除新增的AI智能定位相关功能，保留css失效启用ai定位的策略；增加当工作流跑通时，回填ai选择的css选择器到工作流
2. 属性下的边框样式不统一，为黑线
3. 后台/前台的弹窗提醒样式显示不全，弹窗在上方，超出了浏览器显示范围
4. 现在的节点样式，没办法从当前节点右侧拖拽连接线
5. 去除冗余的回退策略
6. 保存的时候会弹两个成功
7. 等待的秒数按删除键不能删为0，最小为1，不够人性化
8. 截图节点，保存的文件需要能够被专门存储查看
9. 页面加载不全，疑似滚动加载因素，需要检查
## 新增
1. 工作流暴露触发API，设计服务级鉴权，并对外提供skill的调用方式
